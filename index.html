Получаем главную страницу методом GET в протоколе HTTP 1.0.
От HTTPS 1.1 он отличается тем, что в HTTPS 1.0 заголовок Host не является обязательным.
Также в HTTP 1.0 используются базовые методы (GET, POST, HEAD),
тогда как в HTTP/1.1 добавлены новые методы, такие как PUT, DELETE, OPTIONS, TRACE

Получаем внутреннюю страницу методом GET в протоколе HTTP 1.1;

Определяем размер файла file.tar.gz, не скачивая его.
Метод HEAD в HTTP используется для получения только заголовков ответа без тела сообщения.
Сервер возвращает только заголовки ответа, включая Content-Length, который указывает размер файла.
Сам файл не передается.
Информация содержится в Content-length

4й номер.
Медиатип — это информация о типе содержимого файла, которая передается в заголовке HTTP-ответа Content-Type.
Например, для файла image.png медиатип будет image/png.

Отправляем комментарий на сервер по адресу /index.php.
POST — это HTTP-метод, который используется для отправки данных на сервер.
В отличие от GET, который запрашивает данные, POST передает данные в теле запроса.
Content-Type — это заголовок, который указывает тип данных, передаваемых в теле запроса.
application/x-www-form-urlencoded — это стандартный формат для передачи данных форм в HTTP-запросах.
Content-Length — это заголовок, который указывает размер тела запроса в байтах.
